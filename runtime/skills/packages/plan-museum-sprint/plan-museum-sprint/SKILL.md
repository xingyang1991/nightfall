# Plan Museum Sprint

## Overview

把“去看一个展”压缩成可执行的半日奔赴方案，聚焦一个馆/一个展，给出交通、预约、人流策略、馆内停点与收束，并含 Plan B。

## Scope And Guardrails

- 限制范围为一个馆/一个展，避免扩展为全城展讯或多点打卡。
- 控制时长为 3–5 小时（除非用户明确要 8 小时），保持单次奔赴。
- 优先解决怎么去最值、怎么少排队、怎么收束，不做长篇科普。
- 保持馆内停点 ≤ 6，每点只给 1 句“为什么停”。

## Inputs To Collect

- 询问主题偏好（当代/摄影/建筑/设计/传统/自定义）。
- 询问时间预算（3/5/8 小时，或用户自定义）。
- 询问优先级（少排队 vs 氛围最好）。
- 确认出发城市与目标城市（若未提供）。
- 确认交通方式（高铁/自驾/飞机/地铁等，简选即可）。
- 询问是否已有目标馆/展链接或截图，并优先使用用户提供信息。

## Workflow

### Stage 1: 轻参与候选

1. 先问两个关键：主题 + 时间预算；若缺城市先补城市。
2. 检索官方公告/票务平台/用户链接，确认展期、预约或购票、最晚入场、开放时间与闭馆日。
3. 根据时间预算与交通可达性筛 2–3 个候选。
4. 输出 2–3 张候选卡，并提供轻量调整按钮。

确保候选卡包含：
- 馆/展名称 + 推荐时段。
- 风险提示（预约/排队/最晚入场）。
- 时间可达性一句话（往返是否落在预算内）。

调整按钮：`更近` / `更小馆` / `只看一个点版`。

### Stage 2: 敲定半日方案

1. 用户选定后，补问一句：更在意“少排队”还是“气氛最好”。
2. 按偏好调整时段与入场策略（开门冲/傍晚慢/最后一周等）。
3. 输出最终时间轴 + 馆内停点（≤6）+ 收束点 + Plan B。

确保最终方案包含：
- 时间轴（每段 1–2 句，覆盖出发→到馆→入场→馆内→收束→回程）。
- 馆内停点 ≤ 6，每点 1 句理由。
- 入场策略（预约/购票/排队建议）。
- Plan B（预约失败/人太多时的替代馆或只看一个点版）。
- 如可获取，附展览海报图与到馆地图链接。

可选添加看完不散的二次消化入口（阅读/书店/咖啡）。

## Data And Risk Control

- 核验硬信息：展期、最晚入场、预约/购票、开放时间与闭馆日。
- 若信息不可靠，降级为常设/馆藏版本并标注不确定。
- 若预算不足以覆盖往返，提出同城替代或“只看一个点就走”。
- 若预计人流过高，提前提示并给出降级时段或替代馆。

## Output Templates

候选卡模板：

```text
候选 A｜馆名/展名
推荐时段：
风险：预约/排队/最晚入场
时间可达性：
```

最终方案模板：

```text
你选了：馆名/展名（时段）

时间轴：
- 出发：
- 到馆：
- 入场策略：
- 馆内停点（≤6）：
- 收束点：
- 回程：

Plan B：预约失败或人太多时的替代方案
```

## Quality Checks

- 必须包含时间轴。
- 必须有 Plan B。
- 一次选择（时段/去法）能明显改变方案。
- 保持 3–5 小时结构，除非用户明确要求 8 小时。

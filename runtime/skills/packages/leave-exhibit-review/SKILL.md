# 展后留评

## Overview

把看完的展在 5–10 分钟内折成一句话或票根页，让体验留在自己身上。保持轻参与、不评判、默认私有。

## Scope And Guardrails

- 限制为展后 5–10 分钟的二次消化，不延展为长评或科普。
- 避免价值判断或鸡汤表达，不要求长文；一句也成立。
- 默认私有输出，除非用户明确要分享。
- 若涉及图片，仅使用用户提供或明确允许的图，并说明去除 EXIF。

## Inputs To Collect

- 确认方式：`摘一句` / `写一句` / `拼贴` / `语音`。
- 收集一句话或 3–5 个关键词（可空）。
- 补齐展名、时间、地点（仅在缺失时询问）。
- 询问是否有可用照片，或只要文字。
- 询问可选标签 1–3 个（可空）。

## Workflow

### Stage 1: 轻参与候选

1. 先给 2–4 个方式按钮，并说明 5 分钟内可完成。
2. 对选项给“只需填一格”的极短模板。
3. 若用户不想写，直接走“Plan B：三事实票根”。

模板示例：
- 摘一句：`摘一句：______`
- 写一句：`我带走的是：______`
- 拼贴：`三图：入口 / 最心动 / 离场`
- 语音：`6 秒：______`

Plan B（三事实票根）：
- 时间：____
- 地点：____
- 一个细节：____

### Stage 2: 敲定成品

1. 基于用户选择生成成品：标题 + 一句 + 一图 + 标签（可空）。
2. 标题优先用展名；缺失时用“展后留评”。
3. 若是语音，输出 6 秒脚本（1 句），并可选追加票根页。
4. 若是拼贴但无图，给 3 个图位标题与简短说明，等待用户补图。
5. 若有票根模板或渲染器，优先调用；否则用文本排版完成。

## Content Rules

- 控制一句话长度在 12–28 个中文字符或 80 字符以内。
- 以第一人称、感受或细节为主，不写价值评判。
- 避免过度抽象，至少包含一个具体物件、场景或动作。
- 标签 1–3 个，使用 `#` 前缀。

## Data And Risk Control

- 避免个人隐私信息；必要时提示匿名化。
- 不擅自引用未提供的展讯细节；不确定时不写。
- 如涉及图片，提示去除 EXIF 并仅用于私人保存。

## Output Templates

候选入口：

```text
你想用哪种方式留评？（选一个）
按钮：摘一句 / 写一句 / 拼贴 / 语音
```

票根页：

```text
【票根】展名｜日期｜地点
一句：
图：
标签：#____ #____
```

语音脚本：

```text
6 秒脚本：____
```

## Quality Checks

- 首轮只给 2–4 个方式，不给长清单。
- 用户输入最小化：最多填一个空或给关键词即可。
- 最终输出必须是“标题 + 一句 + 一图/图位 + 标签”。
- 无输入时能直接产出三事实票根。

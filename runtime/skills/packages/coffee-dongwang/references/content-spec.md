# 咖啡懂王 内容规范

用途：本规范用于团队搭建技能内容与体验（不涉及工程实现细节）。输出结构为「候选页（多给选择）→ 定稿页（敲定动作）」。

## 它是什么（目标与边界）

- 把“喝咖啡”当作“调频段”的场景决策。
- 先选频段（提神/低因/久坐/快走/社交/独处），再分组给候选。
- 点选后输出定稿页：点单一句话 + 坐法 + 时长 + 非咖啡 Plan B。

## 目标（可衡量）

- 频段按钮一次选定。
- 首屏输出 12-18 个候选（分组呈现）。
- 定稿页必含：点单一句话、坐法一句话、停留时长建议、非咖啡 Plan B。
- 同频段可做连载：提神刊/低因刊/久坐刊等每周更新。

## 硬筛选（用来淘汰，不靠想象）

- 久坐：不强限时（至少允许 90 分钟）。不满足则淘汰或标高风险。
- 快走：动线顺、外带速度快、出杯流程简单。
- 低因：供应明确（菜单/官方/评论）；否则标“低因不稳”，并提升 Plan B。
- 稳定性优先：不追最好喝，追少翻车（出杯稳定/排队可控）。
- 预算可控：支持按预算重排，超预算明确标注。
- 信息冲突不装懂：标风险并抬高 Plan B。

## 细腻标准（编辑总结，带置信度，不做硬门槛）

- 口感描述只做附赠，不做硬门槛。
- 空间质地（明亮快走/昏暖独处）来自图片+文本共识，带置信度。

## 栏目口吻与句式模板（供生成用）

标题模板（1 行）
- 咖啡专家：给你调个频段
- 今天喝什么：{提神/低因/久坐/快走/社交/独处}
- 一杯就够：{动作+频段}

理由句模板（<=2 句）
- 你要的是{频段}，它的{硬信号}更匹配。
- 出杯更稳定/排队更可控。
- Plan B 不喝咖啡也成立。

Need-to-know 清单句式（<=5 条）
- 点单一句话：{低因拿铁/热美式/冰美式少冰}。
- 坐法一句话：{吧台末端/靠墙/靠门快走位}。
- 停留建议：{25/45/90} 分钟，到点就走。
- 排队超过 10 分钟就切 Plan B。
- 不纠结风味，先把频段调对。

风险短写（候选页每卡 <=1 条）
- ⚠️ 排队波动
- ⚠️ 低因不稳
- ⚠️ 久坐可能限时
- ⚠️ 噪音波动
- ✓ 比较稳

## 搜索方向与调用工具（团队工作说明）

搜索方向（要搜什么）
- 先让用户选频段（一个按钮）。
- 检索咖啡/烘焙/外带窗口等业态，按频段筛选。
- 从菜单/评价/图片线索确认：低因供应、是否限时、外带速度、座位结构。
- 分组输出候选：最稳/更便宜/更像独处（每组 4-6）。

调用工具（用什么搜/怎么验证）
- 频段选择器（按钮）。
- POI 搜索器 + 到达时间估算器。
- 菜单与供应线索提取器（低因/外带等）。
- 评论信号提取器（限时/催/排队/噪音）。
- 图片线索提取器（座位/明暗/桌面）。

## 生成流程（先候选，再定稿）

- 用户点频段按钮。
- 候选页输出 12-18 个分组候选，支持换一组。
- 点选后定稿页给点单脚本 + 坐法 + 时长 + 非咖啡 Plan B。
- 收藏为“我的频段”，形成复访。

## 图片与地图规则（保证可执行与合规）

- 候选页小图用杯子/桌角/灯影；避免营销海报。
- 定稿页把点单短句做大字，方便复制。
- 地图可选：定稿页以导航按钮为主。

## 内部学习笔记（不出现在用户文案里）

- “懂王”要懂场景不懂术语：频段比风味更稳定。
- 分组候选提高选择效率与上瘾度。
- Plan B 强制提供“非咖啡也成立”，降低透支风险。

# 雨天硬筛选与细腻标准

## 硬筛选（用来淘汰，不靠想象）

- 室内可成立：连廊/地铁上盖/大堂/图书馆/商场中庭等优先；室内性不成立则剔除。
- 入口友好：雨棚/地下入口/连廊/临停线索清晰；缺失则降权并标风险。
- 续航稳定：雨天也不易提前关门；不确定则标风险并优先 Plan B。
- 拥挤可控：雨天峰值明显的点必须配更稳 Plan B 与切换条件。
- 车主可用：停车规则/临停线索不清则标风险；无法判断则降权。

## 细腻标准（编辑总结，带置信度，不做硬门槛）

- 干燥感：进去后能立刻把湿气放下（写总结 + 置信度）。
- 狼狈成本低：入口清晰、无需站着等很久（写总结 + 置信度）。
- 雨声友好：能听雨但不被雨淹没（只做总结）。

## 雨天条目复核

- 雨天更容易触发拥挤与规则变化，条目需更频繁复核。
- 信息不确定时：优先降权 + 标风险 + 配 Plan B；避免“假确定”。

## 雨天崩盘切换条件（示例）

- 排队超过 10 分钟就切 Plan B。
- 入口拥堵或无雨棚可用时切 Plan B。
- 可坐位明显不足时切 Plan B。

## 可重排维度（候选页）

- 最少淋雨 / 最久能待 / 车主省事 / 最不排队

# 顺路看个小展 · 生成样例（v0.1）

> 这是一个“从 Stage A 候选 → 用户选择 → Stage B 定稿”的完整样例。  
> 地点/机构名称为占位符，目的是展示**表达方式与流程**。

## Stage A 候选卡（CandidateDeck）

我先给你一组候选，你点一个，我再把它写成“能直接出发”的定稿页。  
（你也可以点开关：更近 / 更安静 / 更稳）

**顺路小展 A**
- 适合你：你只需要停 8 分钟，看一个点
- 风险/别踩：风险：可能临时撤展
- 选择：选它定稿

**地铁站微展 B**
- 适合你：更顺路、几乎不需要绕路
- 风险/别踩：风险：高峰期人多
- 选择：选它定稿

**街边装置 C**
- 适合你：光线好，拍一张就走
- 风险/别踩：风险：下雨会影响体验
- 选择：选它定稿

> 用户选择：第 2 张（或你最顺眼的一张）

### Stage B 定稿（系统输出）

**结局卡**
- 主动作：选中一个候选并出发
- Plan B：不成立就切到兜底
- 为什么对你成立：更稳、更省事。
- 风险旗帜：可能人多 / 营业时间有变（信息不足时自动降级）
- 最后 300 米：最后300米：看提示走。
- 怎么待最舒服：怎么待：坐下就别换。
- 切 Plan B 条件：不舒服就切 Plan B。

**PocketTicket（默认私密）**
- 标题：今晚做了一件小事
- 时间：2026-02-04 21:20
- 我选了什么：选中一个候选并出发
- 印章：🖼️
- 轻反馈：喜欢 / 一般 / 不喜欢（原因：太吵 / 没座 / 太远 / 太贵）

**FootprintEntry（默认仅自己可见）**
- 标题：今晚做了一件小事
- 片段：做完了就收束，没有拉长
- 情绪：修复 / 轻探索 / 低压共处（任选）

# 选个chill的地方

## 概述

用“候选页 → 定稿页”的两段式输出，先给一屏可成立的去处，再把用户选中的地点写成可以直接出发的杂志条目。

## 工作流

### Step 0：极少追问（最多 1–2 个按钮）

只在必要时追问；能默认就默认。

- 若用户未给位置：只问一次“你在哪个城市/区域？”
- 若用户未给出行方式/时间/氛围：默认 `步行 + 30–90 分钟 + 安静坐一会`
- 询问格式示例（做成选择条）：`安静坐一会 / 有点人气 / 需要办公 / 只是透气` + `步行 / 地铁 / 开车`

### Step 1：候选池检索与筛选

目标：10 秒内给出默认 6 个“可成立”候选；若用户点“换一批”，最多再扩展一次，总量不超过 12。

硬条件（不满足就剔除或明显标红）：
- **开着**：现在可进入，且预计能撑过停留时长；营业时间至少两处来源一致才视为硬事实，冲突则标记风险
- **可达**：按用户出行方式在 10–20 分钟可达范围内（可放宽到 30 分钟）
- **坐得住**：有“可停留空间”信号（评论、规则、图片、消费结构）
- **不易翻车**：近期出现“排队/限时/催人/不让电脑/极吵”强信号则降权或标风险
- **预算合适**：人均或最低消费超出范围则剔除或标红

细腻标准（只做软标签 + 置信度，不做一票否决）：
- **边界感**：服务是否让独坐舒适、催不催
- **声场质地**：热闹但不炸 / 安静但不冷
- **光感**：明亮 / 昏暖 / 背光柔
- **坐感**：靠墙更稳 / 吧台末端更安心 / 桌面宽窄

排序优先级：
- 最近验证过（最近评论/更新时间）
- 翻车率低（排队、限时、噪音负面少）
- 可复用（稳定营业、座位稳定）

### Step 2：输出候选页（默认 6 个，可展开到 12）

候选页只给关键信息，别展开长文。

顶部固定按钮：`换一批（最多 1 次） / 更近 / 更安静 / 更稳 / 更便宜`

每张卡片包含：
- 名称 + 距离/到达时间
- “能撑到几点”（若不确定，用“预计可待约 X 分钟”）
- 2 个软标签（带置信度，可用低/中/高）
- 1 个风险标记（使用 `references/content-spec.md` 的 risk_flags）
- 1 张小图（可用合规图；没有则用“纹理封面”占位）

分组展示（3 组 × 2 张）：
- 最稳
- 更安静
- 更有质地

底部：一条“点位分布小地图”。

若用户未选择：只提供“换一批”与轻量追问，不要直接给定稿页。

### Step 3：用户点选后输出定稿页

定稿页内容（克制但可执行）：
- 主地址 + 入口/定位提示
- Plan B：更近或更稳的 1 个备选
- 2 句事实型理由（用硬信息，不做幻想）
- “怎么待最舒服”3 条（使用 `references/content-spec.md` 的 checklist 句式）
- 风险旗帜（最多 2）
- 小地图 + 入口提示
- 1 张图：入口/座位/纹理裁切（能提供就提供，不能就不硬凑）

## 文案与品味规则（仅内部）

- 用“动作标题”替代“形容词标题”
- 用“一个小结界”替代“最好”
- 用“对比句”表达品味：同一需求给两种质地
- 图片优先选“入口、桌角、椅背、灯影”，像图录，不像广告

## 搜索与工具（字段级校验）

使用以下“工具/数据源 → 字段验证”的方式，避免各做各的：

- **POI 搜索器**：名称、地址、距离、到达时间、业态类型
- **营业时间校验器**：营业时间至少两处来源一致才算硬事实；不一致则标记 `⚠️ 营业不稳`
- **评论信号提取器**：排队/限时/催人/噪音/独坐友好/插座/电脑友好；记录最近评论时间作为新鲜度
- **规则与价格检索**：最低消费、人均、限时规则；超预算直接剔除或标红
- **图片线索提取器**：只用于软标签（明暗/拥挤/座位形态/桌面大小），并标注置信度
- **路线估算器**：按出行方式估算可达范围（默认 10–20 分钟，可放宽到 30 分钟）
- **可靠性排序器**：最近验证过 + 低翻车率 + 可复用性优先

# 书里偷闲

## 概览
提供一个“能坐住”的书店式空间方案：先给 2–4 个候选，再敲定 1 个最终方案并附 Plan B，重点解决坐哪里、买不买、怎么不尴尬、满座怎么办。

## 核心原则
- 保持克制：首轮只给 2–4 个候选，不做必逛榜。
- 去消费化：默认不要求购买，只给可选点单。
- 够用细节：写清入口、座位与退场方式。
- 风险透明：对噪音/满座/营业时间不确定要提前说明。

## 工作流程（两段式）
### 1. 轻量询问
- 先问用途：`读书 / 工作 / 发呆`。
- 若缺关键信息，再补问：偏好 `更安静 / 更好坐`、时长 `45/90/120 分钟`、当前位置或可达范围。
- 若用户只说“躲雨/避难”，默认 `发呆 + 更安静 + 90 分钟`，并确认。

### 2. 召回与筛选候选
- 以用户位置/城区为中心检索：`书店 / 阅读空间 / 书吧 / 图书馆阅读区` 等。
- 优先筛选：可久待、桌面匹配用途、噪音可控、座位类型清晰。
- 过滤明显排队或强消费场所；不确定时明确提示。
- 准备至少 1 个 Plan B（图书馆/酒店大堂阅读角）。

### 3. 输出首轮候选卡（2–4 张）
- 每张卡只包含：适合用途 + 坐位建议 + 消费压力提示 + 预计可坐时长 + 拥挤风险。
- 提供轻量调整选项：`更安静 / 更好坐 / 更近 / 换一组`。
- 模板：`[候选] 适合用途｜坐位建议｜消费压力｜可坐时长｜拥挤风险`

### 4. 追加确认并敲定
- 用户选定后补问：`更安静还是更好坐`（若未问到）。
- 输出最终方案：
  - 到达后三条建议：入口 / 选座 / 点单可选。
  - 可选 90 分钟节奏：`落座 10 分钟 → 主任务 50 分钟 → 放空 30 分钟`。
  - Plan B：满座/太吵时的替代点。

## 风控与降级
- 无合适书店：切换到 `图书馆阅读区 / 酒店大堂 / 美术馆阅读角`。
- 信息不确定：明确说明不确定，并把 Plan B 与主选项并列。
- 用户不想出门：给“在家 60 分钟阅读角”微方案（整理一角 + 定时）。

## 输出约束
- 首轮候选固定 2–4 个。
- 最终方案固定 1 个主选项 + 1 个 Plan B。
- 必须明确“可不消费也体面”。

## 检索与链接
- 需要时查询：营业时间、座位类型、噪音/满座评价、是否允许久坐。
- 提供地图定位或导航深链（如果可用）。
